5. Example Input File

Here is the input file the author used to align some data taken on a single crystal of Lead Titanate. The K-edge energy of titanium is specified by the e0 keyword. The file format and output file names are specified on the next line. The list of input filenames follows the files keyword. Note that numeric keys are specified for each data set. 

        e0 = 4966
        format = uwxafs         out = tipb_c.nor
        nor1 = 171      nor2 = 261
        pre1 = -100     pre2=-29
        emax = 134
        files
                tipb_c.xmu  5
                tipb_c.xmu  6
                tipb_c.xmu  7
                tipb_c.xmu  8
        end
                tipb_c.xmu  9
                tipb_c.xmu  10

Normally it is not necessary to specify pre1, pre2, nor1, nor2, and emax. I use them in this example simply as a demonstration of these keywords. The default values will yield acceptable results for most data. The derivatives of the data will not be calculated since the derivative keyword is not specified.
