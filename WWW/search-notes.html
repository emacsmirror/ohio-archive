Notes about the current search engine:

  • If you're not getting any results, please keep in mind that the "main" archive is the (currently rather small) set of Emacs Lisp programs which have been formally submitted to the archive. The "incoming" archive, which is a lot larger, is code that has been collected from various places on the web, and is in most cases out of date. If you're looking for a specific piece of code, your best bet is to search both archives.
  • The search algorithm weights words by their frequency, position in the file, and how large the file itself is.
  • No word stemming is performed; For example, to find information about buffers, you'll probably want to search for both "buffer" and "buffers".

About the search engine:

The search engine is running on an independent machine at my apartment, because the CIS department is a bit edgy about allowing dynamic content on their servers. This might change in the future.

The server machine is a 120MHz Pentium with 64MB of RAM running Debian GNU/Linux (potato). The search engine is written in Common Lisp, using the excellent CMUCL implementation, and the Araneida web server.
